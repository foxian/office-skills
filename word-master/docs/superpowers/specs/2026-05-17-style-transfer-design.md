# Word Document Style Transfer Design

## 1. 概述 (Overview)
本项目旨在扩展 Word Master 的能力，实现“学习一个 Word 文档的样式与格式风格，并将其应用到另一个 Word 文档中”。

基于真实业务场景（如招投标书、合同等），源文档中不仅包含可通过 `Heading 1` 描述的标准段落，也包含大量的**非标准格式**（如全靠手动加粗的“假样式”段落），且极可能包含**结构性页面**（如封面、包含表格的签字页等）。

因此，本设计采用 **内容注入与模板克隆相结合** 的技术路线，而不是单纯的“样式复制”。这不仅保证了正文样式的有效映射，更完美解决了复杂封面、页眉页脚与合同表格的精准保留问题。

## 2. 核心架构：模板克隆与内容注入
为了保证目标文档能够100%继承源文档（模板）的复杂结构，我们打破传统的“读取源格式 -> 写入目标”思路，采用以下两步走的注入法：

### 阶段 1：模板预处理与格式分析
1. **生成模板副本**：直接将用户提供的“源模板文档 (A)”物理拷贝一份作为“输出基座文档 (A')”。这一步使得封面、封底、页眉、页脚、固定表格全部被完美克隆。
2. **分析正文规律**：读取源模板文档，针对其中的**正文示范部分**，提取其“格式指纹”（如字号、加粗、对齐等）。
3. **AI 推理映射**：将这些纯视觉的格式指纹交给大模型，推断其对应的语义角色（如“一级标题”、“普通正文”、“引用”等）。

### 阶段 2：内容抽提与精准注入
1. **清理克隆体**：在“输出基座文档 (A')”中，定位到原有的“示范正文区域”（一般在封面/目录之后，签字页之前），将其内容清空，但保留这部分所在“节(Section)”的页面设置。
2. **读取目标内容**：从“目标文档 (B)”（用户实际要写的内容草稿）中提取所有段落的纯文本。
3. **AI 语义套用**：通过大模型判断“目标文档 (B)”中每句话的语义角色，并对照阶段 1 产生的样式映射规则。
4. **注入执行**：将目标内容一行行写入“输出基座文档 (A')”被清空的区域，同时严格应用对应的真样式。

## 3. 具体实现模块 (Modules)

### 3.1 格式分析器 (`style_analyzer.py`)
一个只读的分析脚本。
- **输入**：`source_template.docx`
- **处理**：
  - 遍历所有段落，合并 `paragraph.style` 与 `run-level overrides`，计算出**有效格式指纹**。
  - 去重与聚类，生成 `fingerprints.json`（仅保留代表性段落和示例文字）。
- **输出**：结构化的 JSON 格式指纹摘要。

### 3.2 样式转移总控 (`style_transfer.py`)
这是暴露给用户的入口级脚本，统筹整个工作流。
```bash
python style_transfer.py template.docx draft.docx output.docx
```

它的内部执行逻辑如下：
1. **获取样式配置**：
   - 默认模式：**调用** `style_analyzer.py` 获取 `template.docx` 的指纹摘要，并**调用 LLM** 进行语义推理，生成格式映射配置文件 `style_profile.json`。
   - 复用模式：如果命令行传入了已有的 `.json` 配置文件，则跳过分析和推理阶段，直接使用该配置文件。这使得同一种模板的后续转换速度极快且不需要消耗额外的 tokens。
2. 复制 `template.docx` 到 `output.docx`。
3. **清理与注入**：
   - 识别 `output.docx` 中的“占位/示范正文区”并删除（此步骤的边界识别策略需要在实现阶段细化，可通过固定标记或AI识别进行）。
   - 读取 `draft.docx` 中的内容。
   - 结合上一步获取的配置文件，将内容逐段插入 `output.docx`。

**复用配置的命令行示例：**
```bash
python style_transfer.py --profile style_profile.json template.docx draft.docx output.docx
```

### 3.3 交互与审核模式 (Review Mode)
考虑到完全自动化可能会导致AI推理样式的偏差，提供 `--review` 标志：
```bash
python style_transfer.py template.docx draft.docx output.docx --review
```
系统会在生成 `style_profile.json` 后暂停，等待用户修改确认后，再继续后续的注入写入步骤。

## 4. 边界处理与特殊区域 (封面与签字页)

基于反馈，本次设计的重点限定为：**先处理正文部分的样式转移**。

针对含有**封面、签字承诺函、合同**等特殊格式的页面，由于其排版结构极其特殊（非连续性文本流、绝对定位、大面积留白与下划线等），不能简单采用正文的注入或替换逻辑。

未来的处理策略将是**针对性识别与独立应用**：
1. **识别阶段**：在格式分析时，让 AI 或规则引擎专门识别出这些特殊页面的起止边界（如“前2页为封面，最后1页为签字页”）。
2. **提取模式**：针对封面和签字页，提取其独有的结构化模式（如封面通常由“大字号标题 + 几个带有下划线的表格行”组成）。
3. **针对性应用**：在将草稿内容转换为目标文档时，不是整体注入，而是将草稿中对应的元信息（项目名、日期等）或签字要求，按照提取到的特殊页面模式单独渲染生成。

**本阶段 (Phase 1) 限定**：
当前的 `style_transfer.py` 将**只关注正文段落（标题、正文、列表等段落级别内容）的样式提取与转移**。源模板文档中的封面和签字页在当前版本中暂不作为通用样式被自动转移，以确保核心正文排版逻辑的健壮性。

## 5. 后续实施步骤
1. **编写指纹提取脚本**：开发并独立测试 `style_analyzer.py`，确保其能有效应对假样式并生成紧凑的 `fingerprints.json`。
2. **开发 AI 推理提示词**：设计用于将视觉指纹转化为结构化 JSON profile 的 Prompt，并支持正文区域的语义角色推断。
3. **实现样式转移逻辑**：在 `style_transfer.py` 中，读取源模板的指纹，套用于目标草稿文档（纯文本或初步结构的 Markdown/DOCX），生成格式一致的新文档。
4. **集成与测试**：准备多组含有假样式的正文测试用例，验证端到端效果。
